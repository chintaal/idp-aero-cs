# Deliverables and Expected Outcomes

## 1. Expected Deliverables

### D-1 Concept and Research Documentation

- organized literature review,
- problem statement,
- requirements,
- novelty framing,
- and system architecture notes.

### D-2 Simulation Assets

- RocketPy model,
- dataset-generation scripts or workflow,
- uncertainty sweep definitions,
- and control evaluation plots.

### D-3 Mechanical Assets

- CAD model,
- fabricated airbrake prototype,
- structural and aerodynamic justification.

### D-4 Embedded Assets

- electronics architecture,
- sensor integration,
- actuator control loop,
- and logged test data.

### D-5 Prediction and Control Assets

- trained surrogate model,
- baseline comparisons,
- deployment logic,
- and embedded feasibility evidence.

### D-6 Final Demonstration Package

- integrated prototype,
- validation results,
- final presentation materials,
- and report-ready figures.

## 2. Expected Technical Outcomes

### 2.1 Computer Science and Information Science Outcomes

- demonstration of edge inference for a dynamic control problem,
- sensor-informed predictive modeling,
- model selection under embedded constraints,
- and reproducible evaluation on simulated and hardware-linked data.

### 2.2 Aerospace Outcomes

- demonstration of active drag modulation,
- physically informed airbrake design,
- trajectory-aware control reasoning,
- and structured validation through simulation plus hardware testing.

## 3. Stronger Outcome Statements

Instead of only saying the project will "use PyTorch" or "do CFD," the expected outcomes should be stated as engineering achievements:

- a validated relationship between deployment and drag change,
- a predictor that estimates apogee early enough to support control,
- an embedded loop capable of sensing, prediction, and actuation,
- and evidence that predictive drag deployment improves altitude targeting.

## 4. What a Strong Final Demo Looks Like

A strong final demo does not need to prove everything perfectly. It should clearly show:

- the mechanism actuates reliably,
- the controller responds to state changes,
- the predictor behaves plausibly,
- and the system architecture is technically coherent and extensible.

## 5. Academic Value Beyond the Prototype

Even if the first prototype is modest, the project can still produce reusable outputs:

- a simulation dataset pipeline,
- a benchmark suite for predictive apogee estimation,
- a hardware architecture for future experiments,
- and a documented interdisciplinary methodology.
